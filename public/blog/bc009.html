Multi-Localization Setup: Installing Different BC Versions on One Server

Background

In the world of Business Central On-Premise, you may often need to jump between different localizations. For example, while your primary environment is W1, a new customer might require the US version, which contains specific local features and tax logic. To work efficiently across multiple projects, you need to host both versions on your local machine simultaneously.

The Strategy Instead of spinning up multiple virtual machines, we create side-by-side SQL databases and independent BC Service Instances. This lets you switch between W1 and US localizations simply by changing the URL or port.

Overall Steps

Step 1 — Download

Download the specific localization (e.g., US) from Microsoft.

Step 2 — Restore Database

Restore the localized Demo Database via SSMS.

Step 3 — Create BC Instance

Create and configure a new BC Service Instance using PowerShell.

Technical Implementation

BC On-Prem PowerShell SQL Server IIS

Step 1: Download & Extract

Navigate to the Microsoft Support page to download the specific build (e.g., BC27.5 US). Once downloaded, extract the ZIP file to access the installation media.

Step 2: Restore the Localized Database

Open SQL Server Management Studio (SSMS) and follow these sub-steps:

  • Right-click Databases → Restore Database.
  • Locate the .bak file within the extracted installation folder (usually under ProductDVD\SQL Demo Database).
  • Rename the destination database to something distinct, such as BC270US, to avoid conflicts with your existing W1 database.

Step 3: Create New BC Instance via PowerShell

To simplify this, use the generator at BCHelper.Site to build the setup script automatically.

1. Open the Add BC Instance tab
2. Fill in the new instance configuration
  • Platform Version: 270 — using BC275, so the main platform version is 270.
  • Instance Name: BC270US — distinct from the existing BC270 (W1) instance.
  • Auth Type: Windows — default for SSO.
  • Web Server Instance: BC270US — visible in IIS after creation.
  • Database Server: localhost — DB hosted on the same machine.
  • Database Name: BC270US — the restored database name.
  • Management Services Port: 12045
  • Client Services Port: 8046
  • SOAP Services Port: 8047
  • OData Service Port: 8048
  • Dev Service Port: 8049 — note this port when configuring launch.json.
Note: Choose ports not already reserved by Windows or other running instances to avoid conflicts.
3. Generate & copy the script
4. Open PowerShell ISE as Administrator
5. Paste the script and run (F5)

Wait approximately 3–5 minutes for Windows to create the new instance.

Verify the Result

Check Services

Open the Windows Services panel and confirm the new BC service has been added.

Check IIS

Open IIS Manager and confirm the new web instance (BC270US) appears under Sites.

Launch & Compare

Open both instances side by side to confirm the US localization is running independently from W1.

Done! You now have two independent BC instances running on the same server — one W1 and one US — each with its own database, service, and web endpoint. Switch between them by simply changing the port in your browser or launch.json.
