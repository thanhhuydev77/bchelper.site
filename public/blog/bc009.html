Multi-Localization Setup: Installing Different BC Versions on One Server

Background

In the world of Business Central On-Premise (v27x), you may often need to jump between different localizations. For example, while your primary environment is W1, a new customer might require the US version, which contains specific local features and tax logic. To work efficiently across multiple projects, you need to host both versions on your local machine simultaneously.

The Strategy Instead of multiple virtual machines, we will create side-by-side SQL databases and independent BC Service Instances. This allows you to switch between W1 and US localizations simply by changing the URL or Port.

Overall Steps

  1. Download the specific Localization (US) from Microsoft.
  2. Restore the localized Demo Database via SSMS.
  3. Create and configure a new BC Instance using PowerShell.

Technical Implementation

BC On-Prem PowerShell SQL Server IIS

Step 1: Download & Extract

Navigate to the Microsoft Support page to download the specific build (e.g., BC27.5 US). Once downloaded, extract the ZIP file to access the installation media.

Step 2: Restore the Localized Database

Open SQL Server Management Studio (SSMS) and follow these sub-steps:

  • Right-click Databases -> Restore Database.
  • Locate the .bak file within the extracted installation folder (usually under ProductDVD\SQL Demo Database).
  • Rename the destination database to something distinct, such as BC270US, to avoid conflicts with your W1 database.

Step 3: Create New BC Instance via PowerShell

To simplify this, we can use the generator at BCHelper.Site to create the setup script.

Configuration Parameters:
  • Instance Name: BC270US (Visible in Windows Services).
  • Web Instance: BC270US (Visible in IIS).
  • Auth Type: Windows.
  • Database Server/Name: localhost / BC270US.

Step 4: Port Management

Ensure you select ports that are not already in use by your W1 instance or Windows services. For example:

Service Recommended Port
Management12045
Client8046
SOAP / OData8047 / 8048
Development8049 (Used in launch.json)

Execution

PowerShell (Admin Mode)
# Example snippet generated from BCHelper.Site
New-NAVServerInstance -ServerInstance "BC270US" -ManagementServicesPort 12045 -ClientServicesPort 8046 ...
Set-NAVServerConfiguration -ServerInstance "BC270US" -KeyName "DatabaseName" -KeyValue "BC270US"
# Run F5 to execute the full script
