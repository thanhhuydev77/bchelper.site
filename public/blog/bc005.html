Performance vs. Maintainability: Dictionary of [JsonObject] in AL

Background

In software development, we often face a tug-of-war between high performance and ease of maintenance. When working with temporary data storage in AL (Business Central), three common approaches emerge: Temporary Tables, Dictionaries, and Lists.

The Dilemma: Efficiency vs. Scalability

Traditional Dictionaries offer exceptional $O(1) lookup time but are restricted to a single Key-Value pair. When requirements change such as needing to store three values instead of one - a simple Dictionary fails, forcing developers into "nested dictionaries" that are hard to maintain.

The Pitfalls of Temporary Tables

While Temp Tables are easy to scale by adding fields, they often lead to:

Semantic Mismatch: If a specific field doesn't exist, developers often "borrow" unrelated fields (e.g., using "No. Series" to store a "Bank Account No."), making code confusing.

Performance Overhead: Inserting into tables with many fields is slower than simple memory structures.

Debugging Difficulty: Inspecting large sets of temporary records can be less intuitive than a clear key-value structure.

The Solution: Dictionary of [JsonObject]

To bridge the gap between performance and flexibility, using a Dictionary of [JsonObject] is the most balanced approach.

Flexibility: JsonObject doesn't require a predefined schema. You can add meaningful keys on the fly.

Performance: It retains $O(1)$ search speed. While slightly slower than primitive dictionaries, it is significantly faster than Lists or complex Temp Tables.

Readability: Retrieve values using actual names (e.g., Object.Get('BankAccountNo')) instead of guessing why unrelated fields were reused.

Real-World Benchmark

I processed 500,000+ Customer Ledger Entries to identify unique customers and store metadata.

Test 1: Initial Requirement (Get Unique Customers)
  • Dictionary: Fastest (Pure memory $O(1)$).
  • Temp Record: Fast, but carries table overhead.
  • List: Slowest ($O(n)$ search time).

Temp Record 

procedure UseTempRecord(): text
var
    TempCustomer: Record Customer temporary;
    CustLedEntry: Record "Cust. Ledger Entry";
    BeginTime: Time;
    EndTime: Time;
begin
    BeginTime := CurrentDateTime.Time;
    if CustLedEntry.FindSet() then
        repeat
            if not TempCustomer.get(CustLedEntry."Customer No.") then begin
                TempCustomer.Init();
                TempCustomer."No." := CustLedEntry."Customer No.";
                TempCustomer.Insert();
            end;
        until CustLedEntry.Next() = 0;
    EndTime := CurrentDateTime.Time;

    exit(StrSubstNo('Got %1 Customer in %2 miliseconds by TempRecord', TempCustomer.Count, EndTime - BeginTime));
end;

List of [Code[20]] 

procedure UseList(): Text
var
    CustLedEntry: Record "Cust. Ledger Entry";
    TempCustomer: List of [Code[20]];
    BeginTime: Time;
    EndTime: Time;
begin
    BeginTime := CurrentDateTime.Time;
    if CustLedEntry.FindSet() then
        repeat
            if not TempCustomer.Contains(CustLedEntry."Customer No.") then
                TempCustomer.Add(CustLedEntry."Customer No.");
        until CustLedEntry.Next() = 0;
    EndTime := CurrentDateTime.Time;

    exit(StrSubstNo('Got %1 Customer in %2 miliseconds by List', TempCustomer.Count, EndTime - BeginTime));
end;

Dictionary of [Code[20],Decimal] 

procedure UseDictionaryBasic(): text
var
    CustLedEntry: Record "Cust. Ledger Entry";
    TempCustomer: Dictionary of [Code[20], Decimal];
    BeginTime: Time;
    EndTime: Time;
begin
    BeginTime := CurrentDateTime.Time;
    if CustLedEntry.FindSet() then
        repeat
            if not TempCustomer.ContainsKey(CustLedEntry."Customer No.") then
                TempCustomer.Add(CustLedEntry."Customer No.", 0);
        until CustLedEntry.Next() = 0;
    EndTime := CurrentDateTime.Time;

    exit(StrSubstNo('Got %1 Customer in %2 miliseconds by Dictionary 1', TempCustomer.Count, EndTime - BeginTime));
end;

Dictionary of [Code[20],JsonObject] 

procedure UseDictionaryJsonObject(): text
var
    CustLedEntry: Record "Cust. Ledger Entry";
    DicCustomer: Dictionary of [Code[20], JsonObject];
    BeginTime: Time;
    EndTime: Time;
    JsonObject: JsonObject;
begin
    BeginTime := CurrentDateTime.Time;
    if CustLedEntry.FindSet() then
        repeat
            if not DicCustomer.ContainsKey(CustLedEntry."Customer No.") then
                DicCustomer.Add(CustLedEntry."Customer No.", JsonObject.Clone().AsObject());
        until CustLedEntry.Next() = 0;
    EndTime := CurrentDateTime.Time;

    exit(StrSubstNo('Got %1 Customer in %2 miliseconds by Dictionary JsonObject', DicCustomer.Count, EndTime - BeginTime));
end;

Test 2: Evolving Requirements (Add "Name" & "Bank Account No.")

When adding multiple values, Standard Dictionaries failed. The Temp Table approach required "dirty" field mapping. In contrast, the Dictionary of JsonObject remained clean.

Temp Record 

procedure UseTempRecord2(): text
var
    TempCustomer: Record Customer temporary;
    CustLedEntry: Record "Cust. Ledger Entry";
    BeginTime: Time;
    index: Integer;
    EndTime: Time;
    Name: Text;
    RecipientBankAccount: Text;
begin
    if CustLedEntry.FindSet() then
        repeat
            if not TempCustomer.get(CustLedEntry."Customer No.") then begin
                TempCustomer.Init();
                TempCustomer."No." := CustLedEntry."Customer No.";
                TempCustomer."Name" := CustLedEntry."Customer Name";
                TempCustomer."No. Series" := CustLedEntry."Recipient Bank Account"; //Recipient Bank Account
                TempCustomer.Insert();
            end;
        until CustLedEntry.Next() = 0;

    BeginTime := CurrentDateTime.Time;
    for index := 1 to 1000 do begin
        TempCustomer.Get('CUST' + Format(index));
        Name := TempCustomer."Name";
        RecipientBankAccount := TempCustomer."No. Series";
    end;
    EndTime := CurrentDateTime.Time;

    exit(StrSubstNo('Got %1 Customer in %2 miliseconds by TempRecord', TempCustomer.Count, EndTime - BeginTime));
end;

Dictionary of [Code[20], JsonObject] 

procedure UseDictionaryJsonObject2(): text
var
    CustLedEntry: Record "Cust. Ledger Entry";
    DicCustomer: Dictionary of [Code[20], JsonObject];
    BeginTime: Time;
    EndTime: Time;
    JsonObject: JsonObject;
    Name: Text;
    RecipientBankAccount: Text;
    index: Integer;
begin
    if CustLedEntry.FindSet() then
        repeat
            if not DicCustomer.ContainsKey(CustLedEntry."Customer No.") then begin
                DicCustomer.Add(CustLedEntry."Customer No.", JsonObject.Clone().AsObject());
                DicCustomer.Get(CustLedEntry."Customer No.").Add('Name', CustLedEntry."Customer Name");
                DicCustomer.Get(CustLedEntry."Customer No.").Add('RecipientBankAccount', CustLedEntry."Recipient Bank Account");
            end;
        until CustLedEntry.Next() = 0;

    BeginTime := CurrentDateTime.Time;
    for index := 1 to 1000 do begin
        Name := DicCustomer.Get('CUST' + Format(index)).GetText('Name');
        RecipientBankAccount := DicCustomer.Get('CUST' + Format(index)).GetText('RecipientBankAccount');
    end;
    EndTime := CurrentDateTime.Time;

    exit(StrSubstNo('Got %1 Customer in %2 miliseconds by Dictionary JsonObject', DicCustomer.Count, EndTime - BeginTime));
end;

Conclusion

While Temp Tables are a standard tool, they can lead to "code debt". The Dictionary of JsonObject provides:

  • High Speed: Near-native dictionary performance.
  • Clean Code: Fields are named according to their purpose.
  • Future-Proofing: New data points don't require architectural changes.

For complex logic where speed is critical but requirements evolve, this is the superior choice.
