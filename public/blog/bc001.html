View Attachment File Related to Sales Order on Archived Sales Order

Background

In the company, operations are divided between two primary teams:

  • Sales Order Processing Team: Can create, add attachments, post Sales Orders (SO), and view Archived SOs.
  • Accounting Team: Can view Posted Sales Invoices.
The Problem: In Business Central (BC), a Sales Order is archived and deleted after full posting. The Sales Team can only review the order on the Archived Sales Order page, but standard BC functionality does not include the attachment part on this page.

Requirement

The Sales Team requires the ability to view attachments associated with a fully posted Sales Order via the Archived Sales Order page.

Solution Overview

Add a new Factbox on the Archived Sales Order page. When a user opens the page, the system automatically retrieves attachments related to the Sales Order.

  • Scenario A (Order Exists): If the Sales Order still exists, filter attachments based on the current Sales Order.
  • Scenario B (Order Posted): If the Sales Order is posted, use the Order No. from the Archived SO to find the last Posted Sales Invoice and filter attachments based on that invoice.

Technical Implementation

Factboxes OnAfterGetCurrRecord CurrPage Page.SetTableView Page.Update

Step 1: Add New Factbox to Page Archived Sales Orders

pageextension 70001 "ALE Sales Order Archives" extends "Sales Order Archives"
{
    layout
    {
        addfirst(factboxes)
        {
            part(ALEAttachment; "Doc. Attachment List Factbox")
            {
                ApplicationArea = All;
            }
        }
    }
}

Step 2: Add Function to Filter Related Attachment

local procedure HandleDocumentAttachmentFactbox()
var
    DocumentAttachment: Record "Document Attachment";
begin
    GetAttachmentDocumentFilterFromArchivedSO(Rec, DocumentAttachment);
    CurrPage.ALEAttachment.Page.SetTableView(DocumentAttachment);
    CurrPage.ALEAttachment.Page.Update();
end;

local procedure GetAttachmentDocumentFilterFromArchivedSO(SalesHeaderArchive: Record "Sales Header Archive";
                                                    var DocumentAttachment: Record "Document Attachment")
var
    SalesHeader: Record "Sales Header";
    SalesInvoiceHeader: Record "Sales Invoice Header";
    IsOrderExisted: Boolean;
begin
    IsOrderExisted := SalesHeader.Get(SalesHeaderArchive."Document Type", SalesHeaderArchive."No.");
    
    if IsOrderExisted then begin
        // Case: Order still exists
        DocumentAttachment.SetRange("Table ID", Database::"Sales Header");
        DocumentAttachment.SetRange("Document Type", SalesHeaderArchive."Document Type");
        DocumentAttachment.SetRange("No.", SalesHeaderArchive."No.");
    end else begin
        // Case: Order is fully posted
        SalesInvoiceHeader.SetRange("Order No.", SalesHeaderArchive."No.");
        if SalesInvoiceHeader.FindLast() then begin
            DocumentAttachment.SetRange("Table ID", Database::"Sales Invoice Header");
            DocumentAttachment.SetFilter("No.", SalesInvoiceHeader."No.");
        end;
    end;
end;

Step 3: Add Function to Trigger OnAfterGetCurrRecord

trigger OnAfterGetCurrRecord()
begin
    HandleDocumentAttachmentFactbox();
end;

Pros

  • ✅ No Data Duplication: Does not require duplicating the attachment file itself.
  • ✅ Minimal Customization: Logic is contained within a page extension and helper functions.
  • ✅ Standard Integrity: Keeps standard functionalities intact.

Test Plan

  1. Create a new Sales Order: Create a new Sales Order (SO) and attach a file to it.
  2. Verify Archive (Pre-Post): Click "Archived Document" on the SO, navigate to "Archived Sales Order". You should see the new Document part on the right panel including the file attached in Step 1.
  3. Post Order: Post the full Sales Order. Navigate to "Posted Sales Invoice". The attachment should transfer from the SO automatically (standard BC behavior).
  4. Verify Archive (Post-Post): Navigate to "Archived Sales Order". The attachment should still exist and be visible on the right panel (fetched via the posted invoice link).
